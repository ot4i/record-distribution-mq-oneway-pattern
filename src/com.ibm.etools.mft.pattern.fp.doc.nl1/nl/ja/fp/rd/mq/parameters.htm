入力レコードの処理
パターン・パラメーター M C デフォルト 説明
レコードの検出 いいえ いいえ 固定長 (バイナリーまたはテキスト)

選択されたレコード検出方式に従って、ファイルからレコードが読み取られます。

有効な値は以下のとおりです。

  • 固定長 (バイナリーまたはテキスト)
  • 区切り (バイナリーまたはテキスト)
  • 区切り (XML)
  • 解析済み (バイナリーまたはテキスト)
  • 解析済み (XML)

「解析済み (バイナリーまたはテキスト)」を選択した場合は、「入力メッセージ・セット」、「入力メッセージ・タイプ」、および「入力メッセージ・フォーマット」パラメーターも構成する必要があります。

レコード長 いいえ いいえ 80

必要な長さのレコードがソース・ファイルから読み取られます。
区切り文字タイプ いいえ いいえ DOS または UNIX の行終了

データでデフォルトが使用されない場合、「カスタム区切り文字 (16 進数)」を選択します。
レコード区切り文字 いいえ いいえ  

ファイルから読み取られるレコードは、16 進数として指定される区切り文字によって区切られます。

この区切り文字は、各文字が 0-9 または A-F (大文字または小文字) の間の、最大で 32 文字の偶数の文字として指定する必要があります。

入力メッセージ・セット いいえ いいえ  

このメッセージ・セット内の定義に基づいて、ファイルからレコードが読み取られます。

入力メッセージ・タイプ いいえ いいえ  

このメッセージ・タイプに従って、レコードが読み取られて構文解析されます。

入力メッセージ・フォーマット いいえ いいえ  

このメッセージ・フォーマットに従って、レコードが読み取られて構文解析されます。

ファイルの CCSID はい はい ブローカー・システム・デフォルト

このパラメーターは、読み取られるファイルのバイトを解釈するために使われるコード化文字セットを定義します。
ファイルのエンコード方式 いいえ はい 決定されたブローカー・システム

このパラメーターは、読み取られるファイルのバイトを解釈するために使われる数字およびラージ文字のエンコード・スキームを定義します。 有効な値は以下のとおりです。

  • 決定されたブローカー・システム
  • IEEE 浮動小数点によるリトル・エンディアン (546)
  • IEEE 浮動小数点によるビッグ・エンディアン (273)
  • S390 浮動小数点によるビッグ・エンディアン (785)

エンコード方式については、IBM Integration 資料の『メッセージ・フローを使ったデータ変換』を参照してください。

デフォルト・キュー いいえ はい DEFAULT

このパラメーターは、「経路を指定」プロパティー値の基準が満たされない場合にメッセージの送信先となるキューを定義します。

以下のネーム・スペース・テーブルのパラメーターは、経路のキーの位置を定義する XPath 式で使用できる、ネーム・スペース接頭部およびネーム・スペース URI を定義します。行が含まれる場合、両方のパラメーターを含める必要があります。

テーブルでは、値を追加、編集、および削除したり、値の順序を変更したりすることができます。

ネーム・スペース・テーブルのパラメーター M C デフォルト 説明

ルーティング・テーブルのパラメーター M C デフォルト 説明
キーの位置 はい いいえ

キーの位置は、伝搬されるレコード内の位置を定義する XPath 式です。例えば、$Root/XMLNSC/submitPORequest/partNo というものです。

使用する式のタイプについて詳しくは、IBM Integration 資料の『Route ノード』および『XPath の使用』の項を参照してください。

オプションの「ネームスペース・テーブル」で指定されている場合には、ネーム・スペースを含めることができます。

キー値 はい いいえ

経路の検索
パターン・パラメーター M C デフォルト 説明
ルーティング・データ・ソース いいえ はい

このパラメーターは、経路の選択元にできるデータ・ソースを定義します。 ルーティング情報は ROUTING_TABLE テーブルで検索されます。
ルーティング・データ・コンテキスト いいえ いいえ このパラメーターは、複数のメディエーション・フローで ROUTING_TABLE を使用できるように、ルーティング・コンテキストを定義します。

ルーティング検索で選択され、メモリー内のキャッシュに追加されるのは、ROUTING_CONTEXT フィールドが「ルーティング・データ・コンテキスト」の値と一致する行のみです。

キーの位置 1 いいえ いいえ

このパラメーターは、伝搬されるメッセージ・エレメントから最初のキーを抽出する方法を定義します。 これは ESQL 式として定義されます。

最初のキーは、データ・ソース・ルーティング項目の VARIABLE1 での選択に使用されます。場所は、InputBody または InputRoot の形式でなければなりません。
キーの位置 2 いいえ いいえ

このパラメーターは、オプションの 2 番目のキーを伝搬メッセージ・エレメントから抽出する方法を定義します。 これは ESQL 式として定義されます。

2 番目のキーは、データ・ソース・ルーティング項目の VARIABLE2 での選択に使用されます。場所は、InputBody または InputRoot の形式でなければなりません。
デフォルトのキュー・マネージャー いいえ はい

このパラメーターは、「経路の検索」プロパティー値の基準が満たされない場合にメッセージの送信先となるキュー・マネージャーを定義します。
デフォルト・キュー いいえ はい DEFAULT

このパラメーターは、「経路の検索」プロパティー値の基準が満たされない場合にメッセージの送信先となるキューを定義します。
キャッシュ最新表示 いいえ いいえ クリア済み

このパラメーターを選択した場合、最新表示メッセージを受け取ったときに、ルーティング・データのメモリー内キャッシュがデータ・ソースから最新表示されます。
キューの最新表示 いいえ はい REFRESH

「キャッシュ最新表示」が選択されている場合、このキューに送られたメッセージは、メモリー内のルーティング・データの最新表示を起動します。
ロギング
パターン・パラメーター M C デフォルト 説明
必要なロギング いいえ いいえ クリア済み

このパターン・パラメーターは、ロギング用のコードをパターン・インスタンスに含めるかどうかを決定します。

「必要なロギング」が非選択の場合、ログ・サブフローは含まれません。

「必要なロギング」が選択されている場合、ログ・サブフローがメッセージ・フローに含まれ、「ログ・キュー・マネージャー」と「ログ・キュー」の値を設定する必要があります。
ログ・キュー・マネージャー いいえ はい

このパターン・パラメーターは、ロギング用のキュー・マネージャーを定義します。 ロギングにブローカー・キュー・マネージャーが使用される場合、これをブランクのままにすることができます。

このパラメーターは、「必要なロギング」が選択されている場合にのみ必要です。

ログ・キュー いいえ はい LOG

このパターン・パラメーターは、ロギング用のキューを定義します。

このパラメーターは、「必要なロギング」が選択されている場合にのみ必要です。

その他
パターン・パラメーター M C デフォルト 説明
ブローカー・スキーマ いいえ いいえ mqsi

ブローカー・スキーマを指定すると、すべてのメッセージ・フローと ESQL ファイルがそのスキーマ内に作成されます。 スキーマを指定しない場合、デフォルトのブローカー・スキーマが使用されます。
フロー接頭部 いいえ いいえ

設定されている場合、この接頭部はすべてのメッセージ・フロー名の先頭に追加されます。
フロー接尾部 いいえ いいえ

設定されている場合、この接尾部はすべてのメッセージ・フロー名の末尾に追加されます。
キュー接頭部 いいえ いいえ

設定されている場合、この接頭部はすべてのキュー名の先頭に追加されます。

デフォルトのキュー名が使用されている場合、各パターン・インスタンスのキュー名を識別するために接頭部、接尾部、またはその両方を設定する必要があります。

キュー接尾部 いいえ いいえ

設定されている場合、この接尾部はすべてのキュー名の末尾に追加されます。

デフォルトのキュー名が使用されている場合、各パターン・インスタンスのキュー名を識別するために接頭部、接尾部、またはその両方を設定する必要があります。

簡略説明 いいえ いいえ

この説明は、生成されるすべてのメッセージ・フローの「簡略説明」プロパティーに追加されます。

詳細説明 いいえ いいえ

この説明は、生成されるすべてのメッセージ・フローの「詳細説明」プロパティーに追加されます。

すべてのパターン・パラメーターと、そのデフォルト値がリストされています。

M という見出しの列は、プロパティーが必須 かどうかを示します。C という見出しの列は、プロパティーが構成可能かどうか、つまり、メッセージ・フローをブローカー・アーカイブ (BAR) ファイルに追加するときに値を変更できるかどうかを示します。

デフォルトが示されていない場合、デフォルト値は空ストリングです。
