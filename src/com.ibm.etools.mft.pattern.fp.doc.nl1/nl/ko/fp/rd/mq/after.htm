WebSphere MQ에 대한 레코드 분배: 단방향 패턴을 생성한 후 완료할 태스크

해당 패턴을 생성한 후 완료해야 하는 태스크입니다.

  1. 브로커 아카이브(BAR) 파일을 작성하십시오. 메시지 플로우가 브로커 아카이브 파일에 추가되어야 합니다. 메시지 세트를 사용하여 레코드의 구문 분석 방법을 정의한 경우 해당 메시지 세트를 BAR 파일에 추가해야 합니다.
  2. 브로커 아카이브 파일을 구성하십시오. WebSphere MQ에 대한 레코드 분배: 단방향 패턴 인스턴스를 위한 메시지 플로우를 배치하기 전에 브로커 아카이브 파일에서 메시지 플로우를 추가 구성할 필요는 없지만, 몇 가지 기본 옵션을 검토하고 사용자 환경에 맞게 변경할 수 있습니다.

    브로커 아카이브 편집기의 관리 페이지에서:

    • 선택사항: LoggingOn 사용자 정의 특성을 사용하여 메시지 플로우에 대한 로그 기록의 사용 여부를 선택하십시오.
    • 선택사항: ErrorLoggingOn 사용자 정의 특성을 사용하여 오류 메시지의 생성 여부를 선택하십시오.
    • 선택사항: 브로커 아카이브에서 구성 가능한 특성을 변경할 수도 있습니다(예: 보안 또는 추가 인스턴스 작성에 대한 특성). 값을 변경해도 패턴 인스턴스의 작동에 아무런 영향이 없도록 확인해야 합니다.
  3. 요약 파일에서 작성해야 하는 큐 목록을 확인하십시오. 요약 파일은 InstanceName 패턴 인스턴스 프로젝트의 패턴 구성 폴더에 있습니다.
  4. 큐를 작성하십시오. 패턴을 적용하면 큐 이름과 큐 관리자 이름을 포함한 WebSphere MQ 큐 목록이 생성되며, WebSphere MQ 큐가 아직 없으면 WebSphere MQ 큐를 작성해야 합니다. 패턴에 필요한 모든 메시지 설정은 메시지에 명시적으로 설정되므로 큐 설정에 종속되지 않습니다.
  5. WebSphere MQ 자원을 작성하십시오. 명령행에서 runmqsc와 같은 유틸리티와 프로그램을 사용하거나 WebSphere MQ 탐색기 그래픽 인터페이스를 사용하여 WebSphere MQ 자원을 작성할 수 있습니다. WebSphere MQ 자원 작성에 대한 자세한 정보는 WebSphere MQ 라이브러리 웹 페이지에 있는 WebSphere MQ Intercommunication을 참조하십시오.
  6. 라우팅 매개변수에 라우트 찾아보기 값을 사용하는 경우, 데이터베이스에서 동적으로 라우트를 정의할 수 있지만 해당 라우트가 요약 파일에 나열되지는 않습니다. 데이터베이스에 라우트를 추가한 경우에는 해당 큐 및 큐 관리자를 작성해야 합니다.
  7. 필요하면 라우팅 데이터베이스를 설정하십시오. 라우트 검색 값이 Routing 매개변수에 선택되어 있으면, ROUTING_TABLE 테이블을 사용하여 데이터 소스를 설정해야 합니다. 이 테이블에는 다음과 같은 열이 있어야 합니다.
    • ROUTING_CONTEXT

      플로우의 라우팅 입력 항목을 식별하는 값을 사용합니다.

    • VARIABLE1

      메시지 요소의 KEY1 특성이 일치해야 하는 값입니다.

    • VARIABLE2

      메시지 요소의 KEY2 특성이 일치해야 하는 값입니다.

    • QUEUE_MANAGER

      목적지의 큐 관리자입니다.

    • QUEUE_NAME

      목적지 큐입니다.

  8. FTP를 통해 입력 파일을 얻도록 선택한 경우 패턴에 지정된 이름과 일치하는 대상 브로커에 FtpServer 구성 가능 서비스를 작성해야 합니다. 이 조치는 FileInput 노드에서 원격 FTP 서버에 연결할 때 필요합니다.
  9. BAR 파일을 배치하십시오. 레코드를 정의할 때 메시지 세트를 사용한 경우, 동일한 BAR 파일 또는 다른 BAR 파일을 사용하여 이 메시지 세트를 배치해야 합니다.
