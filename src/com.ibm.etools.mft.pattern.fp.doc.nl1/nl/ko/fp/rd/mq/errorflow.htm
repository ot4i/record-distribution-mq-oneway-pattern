WebSphere MQ에 대한 레코드 분배: 단방향 패턴의 오류 서브플로우

이 서브플로우는 오류 메시지가 기록되어야 하는지 여부를 확인하며, 이는 사용자 정의 특성 ErrorLoggingOn에서 판별합니다.

메시지 플로우에서 처리하는 동안 오류가 포착되면, 오류 서브플로우가 호출됩니다. 오류 서브플로우에서는 다음과 같은 조치를 수행합니다.

  1. 오류 메시지가 필요하면, 오류 메시지를 준비하고 오류 큐에 기록됩니다. 예외 세부사항이 XML 메시지로 기록됩니다. 해당 세부사항은 다음과 같습니다.
    • 브로커 이름
    • 플로우 이름
    • 시간소인
    • 파일 디렉토리
    • 파일 이름
    • 레코드 번호(오류 발생 시)
    • 기본 큐에 기록된 레코드 수
    • 예외 데이터 요약

    예외 데이터 요약에는 메시지 플로우 레이블, 오류 번호, 텍스트 설명 및 추가 세부사항을 제공하는 삽입 목록이 포함되어 있습니다.

  2. 플로우를 비정상적으로 종료시키는 예외가 생성됩니다. 이 조치에 따라 파일이 백아웃 서브디렉토리로 이동합니다.
참고: 라우팅할 수 없는 메시지는 기본 목적지에 기록됩니다. 이 조치는 오류로 처리되지 않습니다.

파일에서 레코드를 읽고 디스크에 기록을 중지시키는 실패는 오류로 처리됩니다.
