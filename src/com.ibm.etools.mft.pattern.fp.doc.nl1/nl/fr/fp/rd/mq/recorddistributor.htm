Flux Distributeur d'enregistrements pour le modèle Distribution d'enregistrements vers WebSphere MQ : unidirectionnel

Informations sur le flux Distributeur d'enregistrements du modèle Distribution d'enregistrements vers WebSphere MQ : unidirectionnel.

Le flux exécute les actions suivantes :

  1. Un fichier est sélectionné pour le traitement en fonction des paramètres d'entrée de fichier.
  2. Les enregistrements sont lus en fonction des paramètres de détection des enregistrements. Chaque enregistrement est traité dans une transaction. Si la validation est spécifiée, l'enregistrement est validé par rapport à un ensemble de messages.
  3. Le flux secondaire Route est appelé pour déterminer la destination WebSphere MQ correcte et l'enregistrement est écrit dans la file d'attente cible appropriée. Lorsqu'un enregistrement est écrit dans une file d'attente, par défaut, son format est identique à celui dans lequel il a été lu dans le fichier. Si nécessaire, vous pouvez convertir l'enregistrement dans un autre format en personnalisant le flux Processeur d'enregistrements.
  4. Si vous avez sélectionné Aucun routage, l'enregistrement est écrit dans la file d'attente par défaut.
  5. Si des erreurs se produisent, par exemple, une incapacité à lire, ou enregistrer ou écrire un enregistrement dans une file d'attente, un message d'erreur est écrit et le flux est annulé. Le fichier est écrit dans le répertoire d'annulation.
  6. Lorsque toutes les données sont traitées, le flux secondaire Journal est appelé pour enregistrer la fin. Si des enregistrements ont été écrits dans la file d'attente des messages par défaut, cette action est consignée.
  7. Le flux aboutit et le fichier est transféré vers le sous-répertoire d'archivage.
