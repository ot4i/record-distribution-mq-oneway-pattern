Flux secondaire Journal du modèle Distribution d'enregistrements vers WebSphere MQ : unidirectionnel

La consignation est contrôlée par la propriété LoggingOn définie par l'utilisateur.

Les messages de journal sont écrits comme messages persistants WebSphere MQ dans une file d'attente WebSphere MQ. Ces messages de journal peuvent être traités séparément en fonction des pratiques standard de votre entreprise.

Les informations suivantes sur le contexte sont situées dans l'en-tête MQRFH2 qui est écrit dans la file d'attente de journaux :

  • Nom du courtier
  • Nom du flux de message
  • Horodatage

Les informations suivantes sur le fichier et les enregistrements se trouvent dans le corps XML du message écrit dans la file d'attente des journaux :

  • Directory
  • Nom de fichier
  • Total d'enregistrements
  • Valeur par défaut des enregistrements

Un message de journal n'est envoyé à la file d'attente de journaux que lorsqu'un fichier entier est terminé avec succès. Tout traitement de fichier générant une erreur et annulant le fichier, générera un message d'erreur contenant les détails de l'emplacement où l'erreur s'est produite.
