Contraintes d'utilisation du modèle Distribution d'enregistrements vers WebSphere MQ : unidirectionnel

Il existe certaines contraintes lorsque vous utilisez ce modèle.

Vous ne pouvez utiliser le modèle Distribution d'enregistrements vers WebSphere MQ : unidirectionnel que si toutes les contraintes suivantes sont respectées :

  • Vous devez pouvoir lire le fichier en entrée sous forme d'enregistrements. Vous pouvez traiter les fichiers qui contiennent des enregistrements de longueur fixe, des enregistrements de longueur variable séparés par un délimiteur identifié ou des enregistrements de longueur variable pouvant être extraits par un analyseur syntaxique, une technique appelée Séquence d'enregistrement analysée. Vous pouvez utiliser cette technique pour lire un fichier qui contient plusieurs documents XML en retournant chaque document sous la forme d'un enregistrement. Vous ne pouvez pas utiliser ce modèle pour extraire des enregistrements imbriqués, tels que des enregistrements dans un document XML.
  • Si un fichier distant est traité par le service FTP, le courtier doit pouvoir supprimer le fichier source, car les fichiers source sont transférés vers le courtier et ne sont pas lus sur place.
  • La fonction métier pour laquelle l'instance de modèle est utilisée doit être compatible avec la redistribution d'enregistrements à la suite d'un incident système. Ce modèle distribue les enregistrements de manière fiable, mais il ne fournit aucune protection contre les doublons après un incident système.
  • La fonction métier pour laquelle l'instance de modèle est utilisée doit être compatible avec la redistribution d'enregistrements à la suite d'un incident afin de lire un enregistrement depuis le fichier ou d'écrire un enregistrement qui n'a pas encore été lu. Dans ce cas, ce modèle déplace la totalité du fichier dans le répertoire d'annulation et une erreur est écrite sur la file d'attente des erreurs. Vous devez faire en sorte que le fichier soit renvoyé une fois l'erreur corrigée. Lorsqu'un fichier est renvoyé, tous les enregistrements du fichier sont retraités, ce qui peut entraîner des messages en double.
  • Si la consignation est nécessaire, le format utilisé par le modèle doit répondre aux conditions de votre environnement.
