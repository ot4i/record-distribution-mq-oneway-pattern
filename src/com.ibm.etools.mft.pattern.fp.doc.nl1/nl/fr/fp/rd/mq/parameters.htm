Traitement d'enregistrement en entrée
Paramètre du modèle O C Valeur par défaut Description
Détection d'enregistrement Non Non Longueur fixe (binaire ou texte)

Les enregistrements sont lus depuis le fichier en fonction de la méthode sélectionnée pour détecter les enregistrements.

Valeurs acceptées :

  • Longueur fixe (binaire ou texte)
  • Délimité (binaire ou texte)
  • Délimité (XML)
  • Analysé (binaire ou texte)
  • Analysé (XML)

Si la valeur Analysé (binaire ou texte) est sélectionnée, les paramètres ensemble de messages en entrée, type de message en entrée, et format de message en entrée doivent également être configurés.
Longueur d'enregistrement Non Non 80

Les enregistrements ayant la longueur appropriée sont lus dans le fichier source.
Type de délimiteur Non Non Fin de ligne DOS ou UNIX

Si vos données n'utilisent pas la valeur par défaut, sélectionnez Délimiteur personnalisé (hexadécimal).
Délimiteur d'enregistrement Non Non  

Les enregistrements lus dans le fichier sont séparés par le délimiteur défini exprimé sous la forme d'une valeur hexadécimale.

Ce délimiteur doit être spécifié sous forme de nombre pair d'un maximum de 32 caractères, dont chacun doit être compris entre 0-9 ou A-F (en majuscule ou minuscule).
Ensemble de messages d'entrée Non Non  

Les enregistrements sont lus à partir d'un fichier basé sur une définition dans cet ensemble de messages.
Type de message d'entrée Non Non  

Les enregistrements sont lus et analysés en fonction de ce type de message.
Format de message d'entrée Non Non  

Les enregistrements sont lus et analysés en fonction de ce format de message.
CCSID du fichier Oui Oui Valeur par défaut du système de courtier

Ce paramètre définit le jeu de caractères codés utilisé pour interpréter les octets du fichier à lire.
Codage du fichier Non Oui Système de courtier déterminé

Ce paramètre définit le schéma de codage des nombres et des caractères utilisés pour interpréter les octets du fichier à lire. Valeurs acceptées :

  • Système de courtier déterminé
  • Format Little endian avec virgule flottante IEEE (546)
  • Format Big endian avec virgule flottante IEEE (273)
  • Format Big endian avec virgule flottante S390 (785)

Pour plus d'informations sur le codage, voir "Conversion des données avec les flux de messages" dans la documentation IBM Integration.

Indiquer les routes
Paramètre du modèle O C Valeur par défaut Description
Gestionnaire de files d'attente par défaut Non Oui

Ce paramètre définit le gestionnaire de files d'attente qui reçoit les messages lorsque les critères pour la propriété Spécifier les routes ne sont pas satisfaits.
File d'attente par défaut Non Oui DEFAULT

Ce paramètre définit la file d'attente qui reçoit les messages lorsque les critères de la propriété Indiquer les routes ne sont pas satisfaits.

Les paramètres qui figurent dans la table Espace de nom suivante définissent les préfixes d'espace de nom et les URI d'espace de nom que vous pouvez utiliser dans les expressions XPath qui définissent les emplacements de clés des routes. Si une ligne est incluse, les deux paramètres doivent être inclus.

Vous pouvez ajouter, modifier et supprimer les valeurs et changer l'ordre des valeurs dans la table.

Paramètre de table d'espace de nom O C Valeur par défaut Description
Préfixe d'espace de nom Oui Non

Ce paramètre définit l'étiquette de l'espace de nom.
URI d'espace de nom Oui Non

Ce paramètre définit l'espace de nom de l'étiquette en cours.

La table Espace de nom définit les espaces de nom que vous pouvez utiliser dans les expressions XPath dans la table Routage.

Paramètre de table de routage O C Valeur par défaut Description
Emplacement de clé Oui Non

Un emplacement de clé est une expression XPath qui définit un emplacement dans l'enregistrement propagé. Par exemple, $Root/XMLNSC/submitPORequest/partNo.

Pour plus d'informations sur le type d'expression à utiliser, voir "Noeud de route" et "Utilisation de XPath" dans la documentation IBM Integration.

Vous pouvez inclure des espaces de nom s'ils sont définis dans la table Espace de nom facultative.

Valeur de clé Oui Non

Ce paramètre contient la valeur à laquelle doit correspondre la zone Emplacement de clé dans le message entrant pour sélectionner la route.
Gestionnaire de files d'attente Non Oui

Ce paramètre définit le gestionnaire de files d'attente d'une destination de routage.
File d'attente Oui Oui

Ce paramètre définit la file d'attente d'une destination de routage.
Rechercher les routes
Paramètre du modèle O C Valeur par défaut Description
Source des données de routage Non Oui

Ce paramètre définit la source de données dans laquelle une route peut être sélectionnée. Les informations de routage sont recherchées dans la table ROUTING_TABLE.
Contexte des données de routage Non Non Ce paramètre définit un contexte de routage pour que vous puissiez utiliser une table ROUTING_TABLE pour plusieurs flux de médiation.

Seules les lignes dans lesquelles la zone ROUTING_CONTEXT correspond à la valeur Contexte de routage sont sélectionnées pour la recherche de routes et ajoutées à la mémoire cache interne.
Emplacement de clé 1 Non Non

Ce paramètre définit le mode d'extraction de la première clé de l'élément de message propagé. Il est défini sous la forme d'une expression ESQL.

La première clé est utilisée pour sélectionner VARIABLE1 dans une entrée de routage. L'emplacement doit être sous la forme InputBody ou InputRoot.
Emplacement de clé 2 Non Non

Ce paramètre définit le mode d'extraction de la seconde clé de l'élément de message propagé. Il est défini sous la forme d'une expression ESQL.

La deuxième clé est utilisée pour sélectionner VARIABLE2 dans une entrée de routage source. L'emplacement doit être sous la forme InputBody ou InputRoot.
Gestionnaire de files d'attente par défaut Non Oui

Ce paramètre définit le gestionnaire de files d'attente qui reçoit les messages lorsque les critères de la propriété Consulter les routes ne sont pas satisfaits.
File d'attente par défaut Non Oui DEFAULT

Ce paramètre définit la file d'attente qui reçoit les messages lorsque les critères de la propriété Consulter les routes ne sont pas satisfaits.
Régénérer le cache Non Non Effacé

Si ce paramètre est sélectionné, lorsqu'un message de régénération est reçu, la mémoire cache interne des données de routage est régénérée depuis la source de données.
File d'attente de régénération Non Oui REFRESH

Si vous sélectionnez Régénérer le cache, un message envoyé à cette file d'attente régénère les données de routage dans la mémoire interne.
Consignation
Paramètre du modèle O C Valeur par défaut Description
Consignation nécessaire Non Non Effacé

Ce paramètre indique si l'instance de modèle inclut le code de consignation.

Si Consignation nécessaire est désélectionné, aucun flux secondaire Journal n'est inclus.

Si Consignation nécessaire est sélectionné, un flux secondaire Journal est inclus dans le flux de messages et vous devez définir les valeurs de Gestionnaire de files d'attente de consignation et File d'attente de journaux.
Gestionnaire de files d'attente de journaux Non Oui

Ce paramètre du modèle définit le gestionnaire de files d'attente de la consignation. Vous pouvez ne pas le définir si vous utilisez le gestionnaire de files d'attente du courtier pour la consignation.

Vous devez le définir uniquement si vous sélectionnez Consignation nécessaire.

File d'attente de consignation Non Oui LOG

Ce paramètre définit la file d'attente de consignation.

Vous devez définir ce paramètre uniquement si vous définissez Consignation nécessaire.
Traitement d'erreurs
Paramètre du modèle O C Valeur par défaut Description
Message d'erreur nécessaire Non Non Sélectionné

Ce paramètre de modèle indique si l'application de modèle crée les éléments de flux de messages qui créent les messages d'erreur.

Si vous sélectionnez Message d'erreur nécessaire, vous devez définir des valeurs pour Gestionnaire de files d'attente des erreurs et File d'attente des erreurs.
Gestionnaire de files d'attente d'erreurs Non Oui  

Ce paramètre de modèle définit le gestionnaire de files d'attente des messages d'erreur. Vous pouvez ne pas le définir si vous utilisez le gestionnaire de files d'attente du courtier pour la consignation.
File d'attente d'erreurs Non Oui ERROR

Ce paramètre du modèle définit la file d'attente des messages d'erreur.

Vous devez le définir uniquement si vous sélectionnez Message d'erreur nécessaire.

Dispositions générales
Paramètre du modèle O C Valeur par défaut Description
Schéma de courtier Non Non mqsi

Si un schéma de courtier est défini, tous les flux de messages et fichiers ESQL sont créés dans le schéma. Si aucun schéma n'est défini, le schéma de courtier par défaut est utilisé.
Préfixe de flux Non Non

Si vous définissez ce préfixe, il est ajouté au début de tous les noms de flux de messages.
Suffixe de flux Non Non

Si vous définissez ce suffixe, il est ajouté à la fin de tous les noms de flux de messages.
Préfixe de file d'attente Non Non

Si vous définissez ce préfixe, il est ajouté au début de tous les noms de file d'attente.

Si vous utilisez les noms de file d'attente par défaut, vous devez définir le préfixe, le suffixe ou les deux pour identifier les noms de file d'attente de chaque instance du modèle.

Suffixe de file d'attente Non Non

Si vous définissez ce suffixe, il est ajouté à la fin de tous les noms de file d'attente.

Si vous utilisez les noms de file d'attente par défaut, vous devez définir le préfixe, le suffixe ou les deux pour identifier les noms de file d'attente de chaque instance du modèle.

Description abrégée Non Non

Cette description est ajoutée à la propriété Description abrégée de tous les flux de messages générés.
Description détaillée Non Non

Cette description est ajoutée à la propriété Description détaillée de tous les flux de messages générés.

Tous les paramètres du modèle sont répertoriés avec leur valeur par défaut.

La colonne avec l'en-tête M indique si la propriété est obligatoire. La colonne avec l'en-tête C indique si la propriété est configurable, ce qui signifie que vous pouvez modifier la valeur lorsque vous ajoutez le flux de messages au fichier d'archive du courtier (BAR).

Lorsqu'aucune valeur n'est définie, la valeur par défaut est une chaîne vide.
