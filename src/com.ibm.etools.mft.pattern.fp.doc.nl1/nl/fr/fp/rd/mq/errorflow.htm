Flux secondaire Erreur du modèle Distribution d'enregistrements vers WebSphere MQ : unidirectionnel

Ce flux secondaire teste si un message d'erreur doit être écrit, ce qui est déterminé par la propriété ErrorLoggingOn définie par l'utilisateur.

Si une erreur est interceptée lors du traitement dans le flux de messages, le flux secondaire Erreur est appelé. Le flux secondaire Erreur exécutent les actions suivantes :

  1. Si des messages d'erreur sont requis, un message d'erreur est rédigé et écrit dans la file d'attente des erreurs. Les détails de l'exception sont écrits sous forme de message XML. Ces détails comprennent :
    • Nom du courtier
    • Nom du flux
    • Horodatage
    • Répertoire de fichier
    • Nom de fichier
    • Numéro d'enregistrement, lorsque l'erreur s'est produite
    • Nombre d'enregistrements inscrits dans la file d'attente par défaut
    • Récapitulatif des données de l'exception

    Le récapitulatif des données d'exception contient le libellé du flux de messages, le numéro d'erreur, la description de texte, ainsi qu'une liste d'insertions donnant plus de détails.

  2. Une exception générée provoque l'échec du flux. Cette action transfère le fichier vers le sous-répertoire d'annulation.
Remarque : les messages qui ne peuvent pas être routés sont écrits dans une destination par défaut. Cette action n'est pas traitée comme une erreur.

Tout incident qui empêche la lecture d'un enregistrement dans le fichier et son écriture sur le disque est traité comme une erreur.
