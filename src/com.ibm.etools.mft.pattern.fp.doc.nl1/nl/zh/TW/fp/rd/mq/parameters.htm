Record Distribution to WebSphere MQ: one-way 型樣的參數

輸入檔
型樣參數 M C 預設值 說明
輸入檔目錄 是 是  

輸入檔的位置。
檔案型樣 是 是 *

包含選用萬用字元（星號 (*) 或問號 (?)）的檔名或字串，可識別要從輸入目錄處理的一個以上檔案。
使用 FTP 是 是 已清除

此參數決定是否使用 FTP 來擷取遠端檔案。
FTP 可配置的服務 否 是 ftpservice

這個可配置的服務定義如何使用 FTP 來擷取遠端檔案。
輸入記錄處理
型樣參數 M C 預設值 說明
記錄偵測 否 否 固定長度（二進位或文字）

根據選取的記錄偵測方法，從檔案讀取記錄。

有效值為：

  • 固定長度（二進位或文字）
  • 定界 (二進位或文字)
  • 定界 (XML)
  • 剖析 (二進位或文字)
  • 剖析 (XML)

如果選取剖析（二進位或文字），則也必須配置輸入訊息集、輸入訊息類型及輸入訊息格式參數。
記錄長度 否 否 80

所需長度的記錄是讀取自原始檔。
定界字元類型 否 否 DOS 或 UNIX 行尾

如果您的資料未使用預設值，請選取自訂定界字元（十六進位）。
記錄定界字元 否 否  

讀取自檔案的記錄會使用指定的定界字元（其為十六進位格式）加以區隔。

此定界字元必須以偶數個字元指定，最多 32 個字元，而且每個字元的範圍必須介於 0-9 或 A-F 之間（大寫或小寫都可以）。
輸入訊息集 否 否  

記錄是根據此訊息集中的定義，從檔案中進行讀取。
輸入訊息類型 否 否  

記錄是根據此訊息類型進行讀取及剖析。
輸入訊息格式 否 否  

記錄是根據此訊息格式進行讀取及剖析。
檔案的 CCSID 是 是 分配管理系統預設值

此參數定義編碼字集，以用來解譯所讀取檔案的位元組。
檔案編碼 否 是 由分配管理系統決定

此參數定義數字及大量字元的編碼方法，而數字及大量字元是用來解譯所讀取檔案的位元組。 有效值為：

  • 由分配管理系統決定
  • 小序排列法，含 IEEE 浮點數 (546)
  • 大序排列法，含 IEEE 浮點數 (273)
  • 大序排列法，含 S390 浮點數 (785)

如需編碼的相關資訊，請參閱 IBM Integration 說明文件中的「使用訊息流程轉換資料」。

輸出佇列 否 是 OUTPUT

預設佇列管理程式 否 是

此參數定義不符合指定路徑內容值的準則時，要傳送訊息的目標佇列管理程式。
預設佇列 否 是 DEFAULT

此參數定義不符合指定路徑內容值的準則時，要傳送訊息的目標佇列。

下列名稱空間表格中參數所定義的名稱空間字首及名稱空間 URI，可讓您用來在 XPath 表示式中定義路徑的索引鍵位置。 如果包括列，則必須包括這兩個參數。

您可以新增、編輯及刪除值，以及變更值在表格中的順序。

名稱空間表格參數 M C 預設值 說明
名稱空間字首 是 否

此參數定義名稱空間的標籤。
名稱空間 URI 是 否

此參數定義現行標籤的名稱空間。

名稱空間表格定義遞送表中可讓您用於 XPath 表示式的名稱空間。

遞送表參數 M C 預設值 說明
索引鍵位置 是 否

索引鍵位置是 XPath 表示式，用來定義記錄內正在傳播的位置。例如，$Root/XMLNSC/submitPORequest/partNo。

如需要使用的表示式類型的相關資訊，請參閱 IBM Integration 說明文件中的「遞送節點」與「使用 XPath」。

如果在選用名稱空間表格中指定名稱空間，則可以將其併入。

索引鍵值 是 否

此參數包含的值是送入訊息中索引鍵位置欄位必須與之相符才能選取此路徑的值。
佇列管理程式 否 是

此參數定義遞送目的地的佇列管理程式。
佇列 是 是

此參數定義遞送目的地的佇列。
查閱路徑
型樣參數 M C 預設值 說明
遞送資料來源 否 是

此參數定義可從中選取路徑的資料來源。 遞送資訊是在 ROUTING_TABLE 表格中進行查閱。
遞送資料環境定義 否 否 此參數定義遞送環境定義，讓您可以將 ROUTING_TABLE 用於多個調解流程。

只有 ROUTING_CONTEXT 欄位與遞送資料環境定義值相符的列才會選取來進行遞送查閱，也只有這些列才會新增至記憶體內快取。
索引鍵位置 1 否 否

此參數定義如何從傳播的訊息元素取出第一個索引鍵。它定義為 ESQL 表示式。

第一個索引鍵是用來選取資料來源遞送登錄中的 VARIABLE1。此位置的格式必須為 InputBody 或 InputRoot。
索引鍵位置 2 否 否

此參數定義如何從傳播的訊息元素取出選用的第二個索引鍵。它定義為 ESQL 表示式。

第二個索引鍵是用來選取資料來源遞送登錄中的 VARIABLE2。 此位置的格式必須為 InputBody 或 InputRoot。
預設佇列管理程式 否 是

此參數定義不符合查閱路徑內容值的準則時，要傳送訊息的目標佇列管理程式。
預設佇列 否 是 DEFAULT

此參數定義不符合查閱路徑內容值的準則時，要傳送訊息的目標佇列。
快取重新整理 否 否 已清除

如果選取此參數，則在接收到重新整理訊息時，會從資料來源中重新整理遞送資料的記憶體內快取。
重新整理佇列 否 是 REFRESH

如果選取快取重新整理，則傳送至此佇列的訊息會觸發記憶體內遞送資料的重新整理作業。
記載
型樣參數 M C 預設值 說明
需要記載 否 否 已清除

此型樣參數決定型樣實例是否包括用於記載的程式碼。

如果清除需要記載，則不會包括「日誌」子流程。

如果選取需要記載，則「日誌」子流程會併入訊息流程中，而且必須設定日誌佇列管理程式及日誌佇列的值。
日誌佇列管理程式 否 是

此型樣參數定義佇列管理程式以進行記載。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。

只有在選取需要記載時，才需要此參數。

日誌佇列 否 是 LOG

此型樣參數定義佇列以進行記載。

只有在選取需要記載時，才需要此參數。
錯誤處理
型樣參數 M C 預設值 說明
需要錯誤訊息 否 否 已選取

此型樣參數定義型樣應用程式是否建立可建立錯誤訊息的訊息流程元素。

如果選取需要錯誤訊息，則必須設定錯誤佇列管理程式及錯誤佇列的值。
錯誤佇列管理程式 否 是  

此型樣參數定義錯誤訊息的佇列管理程式。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。
錯誤佇列 否 是 ERROR

此型樣參數定義錯誤訊息的佇列。

只有在選取需要錯誤訊息時，才需要此參數。

一般
型樣參數 M C 預設值 說明
分配管理系統綱目 否 否 mqsi

如果指定分配管理系統綱目，則會在該綱目內建立所有訊息流程及 ESQL 檔案。如果未指定任何綱目，則會使用預設分配管理系統綱目。
流程字首 否 否

如果已設定，則會將此字首新增至所有訊息流程名稱的開頭。
流程字尾 否 否

如果已設定，則會將此字尾新增至所有訊息流程名稱的結尾。
佇列字首 否 否

如果已設定，則會將此字首新增至所有佇列名稱的開頭。

如果使用預設佇列名稱，則必須設定字首及（或）字尾，才能識別每一個型樣實例的佇列名稱。

佇列字尾 否 否

如果已設定，則會將此字尾新增至所有佇列名稱的結尾。

如果使用預設佇列名稱，則必須設定字首及（或）字尾，才能識別每一個型樣實例的佇列名稱。

簡要說明 否 否

此說明會新增至所有產生的訊息流程的簡要說明內容。
詳細說明 否 否

此說明會新增至所有產生的訊息流程的詳細說明內容。

所有型樣參數會與其預設值一起列出。

標頭為 M 的直欄指出內容是否為必要的。 標頭為 C 的直欄指出內容是否為可配置的，這表示您可以在將訊息流程新增至分配管理系統保存檔 (BAR) 時變更值。

如果未指出任何預設值，則預設值是空字串。
