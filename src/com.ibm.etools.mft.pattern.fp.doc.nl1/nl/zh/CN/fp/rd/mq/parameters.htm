Record Distribution to WebSphere MQ: one-way 模式的参数

输入文件
模式参数 M C 缺省值 描述
输入文件目录 是 是  

输入文件的位置。
文件模式 是 是 *

包含可选通配符（星号 (*) 或问号 (?)）的文件名或字符串，通配符可标识要处理的输入目录中的一个或多个文件。
使用 FTP 是 是 已清除

此参数确定是否使用 FTP 来检索远程文件。
FTP 可配置服务 否 是 ftpservice

这一可配置服务确定如何使用 FTP 来检索远程文件。
输入记录处理
模式参数 M C 缺省值 描述
记录检测 否 否 定长（二进制或文本）

根据选中的记录检测方法从文件中读取记录。

有效值为：

  • 定长（二进制或文本）
  • 定界（二进制或文本）
  • 定界（XML）
  • 已解析（二进制或文本）
  • 已解析（XML）

如果已选择已解析（二进制或文本），那么还必须配置输入消息集、输入消息类型和输入消息格式参数。
记录长度 否 否 80

从源文件中读取规定长度的记录。
定界符类型 否 否 DOS 或 UNIX 行终点

如果数据不使用缺省值，请选择定制定界符（十六进制）。
记录定界符 否 否  

从文件中读取的记录以指定的十六进制定界符分隔。

此定界符必须指定为最多不超过 32 个字符的偶数，并且每个字符必须介于 0 到 9 或 A 到 F 之间（大小写均可）。
输入消息集 否 否  

根据此消息集中的定义从文件中读取记录。
输入消息类型 否 否  

根据此消息类型读取和解析记录。
输入消息格式 否 否  

根据此消息格式读取和解析记录。
文件 CCSID 是 是 代理系统缺省值

此参数定义用于解释正在读取的文件字节的编码字符集。
文件编码 否 是 代理系统已确定

此参数定义数字和大字符的编码方案，这些数字和字符用于解释正在读取的文件的字节。 有效值为：

  • 代理系统已确定
  • 带有 IEEE 浮点的小尾数法（546）
  • 带有 IEEE 浮点的大尾数法（273）
  • 带有 S390 浮点的大尾数法（785）

有关编码的信息，请参阅 IBM Integration 文档中的“使用消息流转换数据”。
路由
模式参数 M C 缺省值 描述
路由 否 否 无路由

此参数定义要使用的路由方法。有效值为：

  • 无路由
  • 指定路由
  • 查找路由
无路由
模式参数 M C 缺省值 描述
输出队列管理器 否 是

该参数定义向其发送所有输出消息的队列管理器。
输出队列 否 是 OUTPUT

该参数定义向其发送所有输出消息的队列。
指定路由
模式参数 M C 缺省值 描述
缺省队列管理器 否 是

此参数定义在未满足指定路由属性值的条件时向其发送消息的队列管理器。
缺省队列 否 是 DEFAULT

此参数定义在未满足指定路由属性值的条件时向其发送消息的队列。

以下名称空间表中的参数定义名称空间前缀和名称空间 URI，您可以使用它们在 XPath 表达式中定义路由的键位置。 如果包含行，那么必须包含这两个参数。

您可以添加、编辑和删除值，以及更改值在表中的顺序。

名称空间表参数 M C 缺省值 描述
名称空间前缀 是 否

此参数定义名称空间标记。
名称空间 URI 是 否

此参数定义当前标记的名称空间。

名称空间表定义可在路由表的 XPath 表达式中使用的名称空间。

路由表参数 M C 缺省值 描述
键位置 是 否

键位置是一个 XPath 表达式，该表达式定义正在传播的记录中的某个位置。例如，$Root/XMLNSC/submitPORequest/partNo。

有关要使用的表达式类型的更多信息，请参阅 IBM Integration 文档中的“Route 节点”和“使用 XPath”。

如果名称空间指定于可选的名称空间表，那么可包括这些名称空间。

键值 是 否

此参数包含的值必须与入局消息中的“键位置”字段匹配，才能选择此路由。

队列管理器 否 是

此参数定义路由目标的队列管理器。
队列 是 是

此参数定义路由目标的队列。
查找路由
模式参数 M C 缺省值 描述
路由数据源 否 是

此参数定义可从其中选择路由的数据源。 可在 ROUTING_TABLE 表中查找路由信息。
路由数据上下文 否 否 该参数定义路由上下文，以便您可以将 ROUTING_TABLE 用于多个调解流。

只有 ROUTING_CONTEXT 字段匹配路由数据上下文值的行才能被选择用来进行路由查找，并添加到内存高速缓存。

键位置 1 否 否

此参数定义如何从传播的消息元素中抽取第一个键。该参数定义为 ESQL 表达式。

第一个键用于在数据源路由条目中的 VARIABLE1 上进行选择。位置必须为 InputBody 或 InputRoot 格式。

键位置 2 否 否

此参数定义如何从传播的消息元素中抽取可选的第二个键。该参数定义为 ESQL 表达式。

第二个键用于在数据源路由条目中的 VARIABLE2 上进行选择。 位置必须为 InputBody 或 InputRoot 格式。

缺省队列管理器 否 是

此参数定义在未满足查找路由属性值的条件时向其发送消息的队列管理器。
缺省队列 否 是 DEFAULT

此参数定义在未满足查找路由属性值的条件时向其发送消息的队列。
高速缓存刷新 否 否 已清除

如果选择了此参数，那么在收到刷新消息时，将从数据源刷新路由数据的内存高速缓存。
刷新队列 否 是 REFRESH

如果选择了高速缓存刷新，那么发送到此队列的消息会触发内存路由数据刷新。
记录
模式参数 M C 缺省值 描述
需要记录 否 否 已清除

此模式参数确定模式实例是否包括用于记录的代码。

如果清除了需要记录，那么将不包括日志子流。

如果选择了需要记录，那么消息流中将包括日志子流，并且必须设置日志队列管理器和日志队列的值。
日志队列管理器 否 是

此模式参数定义用于记录的队列管理器。 如果将代理队列管理器用于记录，那么此参数可保留为空。

仅当选择了需要记录时才需要此参数。

日志队列 否 是 LOG

此模式参数定义用于记录的队列。

仅当选择了需要记录时才需要此参数。
错误处理
模式参数 M C 缺省值 描述
需要错误消息 否 否 已选中

此模式参数定义模式应用程序是否创建消息流元素，该元素创建错误消息。

如果选择了需要错误消息，那么必须设置错误队列管理器和错误队列的值。
错误队列管理器 否 是  

此模式参数定义错误消息的队列管理器。 如果将代理队列管理器用于记录，那么此参数可保留为空。
错误队列 否 是 ERROR

此模式参数定义错误消息的队列。

仅当选择了需要错误消息时才需要此参数。

常规
模式参数 M C 缺省值 描述
代理模式 否 否 mqsi

如果指定了代理模式，那么将在该模式中创建所有的消息流和 ESQL 文件。如果未指定模式，那么将使用缺省代理模式。
流前缀 否 否

如果设置了此参数，那么会将此前缀添加到所有消息流名称的开始处。
流后缀 否 否

如果设置了此参数，那么会将此后缀添加到所有消息流名称的结尾处。
队列前缀 否 否

如果设置了此参数，那么会将此前缀添加到所有队列名称的开始处。

如果使用了缺省队列名称，那么必须设置前缀和/或后缀，以标识每个模式实例的队列名称。

队列后缀 否 否

如果设置了此参数，那么会将此后缀添加到所有队列名称的结尾处。

如果使用了缺省队列名称，那么必须设置前缀和/或后缀，以标识每个模式实例的队列名称。

简短描述 否 否

此描述将被添加到生成的所有消息流的简短描述属性。
详细描述 否 否

此描述将被添加到生成的所有消息流的详细描述属性。

列出所有模式参数及其缺省值。

标题为 M 的列指示属性是否为必需的。 标题为 C 的列指示属性是否为可配置的，也就是说，可以在将消息流添加到代理归档 (BAR) 文件时更改值。

未表明缺省值时，缺省值是空字符串。
