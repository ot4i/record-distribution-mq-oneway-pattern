Parâmetros para o padrão de Record Distribution para WebSphere MQ: Unidirecional

Arquivo de entrada
Parâmetro de padrão M C Default Descrição
Diretório do arquivo de entrada Sim Sim  

O local do arquivo de entrada
Padrão do arquivo Sim Sim *

Um nome do arquivo ou cadeia contendo caracteres curingas opcionais (asterisco (*) ou interrogação (?)) identificando o arquivo, ou os arquivos, para processamento a partir do diretório de entrada.
Usar FTP Sim Sim Limpo

Este parâmetro determina se o FTP é usado para recuperar arquivos remotos.
Serviço configurável de FTP Não Sim ftpservice

Este serviço configurável define como o FTP é usado para recuperar um arquivo remoto.
Processamento de registro de entrada
Parâmetro de padrão M C Default Descrição
Detecção de registro Não Não Comprimento fixo (binário ou texto)

Registros são lidos a partir do arquivo de acordo com o método de detecção de registro que é selecionado.

Os valores válidos são:

  • Comprimento fixo (binário ou texto)
  • Delimitado (binário ou texto)
  • Delimitado (XML)
  • Analisado (binário ou texto)
  • Analisado (XML)

Se Analisado (binário ou texto) for selecionado, os parâmetros Conjunto de mensagens de entrada, Tipo de mensagem de entrada e Formato de mensagem de entrada também deverão ser configurados.
Duração do registro Não Não 80

Registros de duração necessária são lidos a partir do arquivo de origem.
Tipo de delimitador Não Não fim da linha DOS ou UNIX

Se seus dados não usarem o padrão, selecione Delimitador customizado (hexadecimal).
Delimitador de registro Não Não  

Registros que são lidos a partir do arquivo são separados pelo delimitador especificado, que é fornecido em hexadecimal.

Este delimitador deve ser especificado como um número par de até 32 caracteres, cada caractere deve ser entre 0-9 ou A-F (maiúscula ou minúscula).
Conjunto de mensagens de entrada Não Não  

Registros são lidos a partir do arquivo com base em uma definição neste conjunto de mensagens.
Tipo de mensagem de entrada Não Não  

Registros são lidos e analisados de acordo com este tipo de mensagem.
Formato da mensagem de entrada Não Não  

Registros são lidos e analisados de acordo com este formato da mensagem.
CCSID de arquivo Sim Sim Padrão do sistema do broker

Este parâmetro define o conjunto de caracteres codificados que é usado para interpretar bytes do arquivo que estão sendo lidos.
Codificação do arquivo Não Sim Determinado pelo sistema do broker

Esta parâmetro define o esquema de codificação para os números e caracteres grandes que são usados para interpretar bytes do arquivo que estão sendo lidos. Os valores válidos são:

  • Determinado pelo sistema do broker
  • Little endian com ponto flutuante IEEE (546)
  • Big endian com ponto flutuante IEEE (273)
  • Big endian com ponto flutuante S390 (785)

Para obter informações sobre codificação, veja "Convertendo Dados com Fluxos de Mensagens" na documentação do IBM Integration.
Roteamento
Parâmetro de padrão M C Default Descrição
Roteamento Não Não Sem roteamento

Este parâmetro define o método de roteamento a usar. Os valores válidos são:

  • Sem roteamento
  • Especificar rotas
  • Consultar rotas
Sem roteamento
Parâmetro de padrão M C Default Descrição
Gerenciador de filas de saída Não Sim

Este parâmetro define o gerenciador de filas para o qual todas as mensagens de saída são enviadas.
Fila de saída Não Sim OUTPUT

Este parâmetro define a fila para a qual todas as mensagens de saída são enviadas.
Especificar rotas
Parâmetro de padrão M C Default Descrição
Gerenciador de filas padrão Não Sim

Este parâmetro define o gerenciador de filas para o qual as mensagens são enviadas quando os critérios para o valor da propriedade Especificar rotas não são atendidos.
Fila padrão Não Sim DEFAULT

Este parâmetro define a fila para a qual mensagens são enviadas quando os critérios para o valor da propriedade Especificar rotas não são atendidos.

Os parâmetros na Tabela de Espaço de Nomes a seguir definem os prefixos de espaço de nomes e os URIs de espaço de nomes que podem ser usados nas expressões XPath que definem os locais de chave para rotas. Se uma linha for incluída, ambos os parâmetros deverão ser incluídos.

É possível incluir, editar e excluir valores e alterar a ordem dos valores na tabela.

Parâmetro da tabela de espaço de nomes M C Default Descrição
Prefixo de espaço de nomes Sim Não

Este parâmetro define a tag para um espaço de nomes.
URI do espaço de nomes Sim Não

Este parâmetro define o espaço de nomes para a tag atual.

A Tabela de Espaço de Nomes define os espaços de nomes que estão disponíveis para você usar nas expressões XPath na Tabela de Roteamento.

Parâmetro da tabela de roteamento M C Default Descrição
Local da chave Sim Não

Um local de chave é uma expressão XPath que define um local no registro que está sendo propagado. Por exemplo, $Root/XMLNSC/submitPORequest/partNo.

Para obter informações adicionais sobre o tipo de expressão a usar, consulte "Nó da Rota" e "Usando XPath" na documentação do IBM Integration.

Os espaços de nomes podem ser incluídos se forem especificados na Tabela de Espaço de Nomes opcional.

Valor da chave Sim Não

Este parâmetro contém o valor que o campo Local da chave na mensagem recebida deve corresponder para selecionar esta rota.
Gerenciador de filas Não Sim

Este parâmetro define o gerenciador de filas para um destino de roteamento.
Fila Sim Sim

Este parâmetro define a fila para um destino de roteamento.
Rotas de Consulta
Parâmetro de padrão M C Default Descrição
Origem de dados de roteamento Não Sim

Este parâmetro define a origem de dados a partir da qual uma rota pode ser selecionada. Informações de roteamento são consultadas na tabela ROUTING_TABLE.
Contexto de dados de roteamento Não Não Este parâmetro define um contexto de roteamento para que você possa usar um ROUTING_TABLE para mais de um fluxo de mediação.

Apenas linhas nas quais o campo ROUTING_CONTEXT corresponde ao valor de Contexto dos Dados de Roteamento são selecionadas para consulta de roteamento e são incluídas no cache na memória.
Local de chave 1 Não Não

Este parâmetro define como a primeira chave é extraída do elemento de mensagem propagado. Ele é definido como uma expressão ESQL.

A primeira chave é usada para seleção em VARIABLE1 em uma entrada de roteamento da origem de dados. O local deve estar no formato InputBody ou InputRoot.
Local de chave 2 Não Não

Este parâmetro define como uma segunda chave opcional é extraída do elemento de mensagem propagado. Ele é definido como uma expressão ESQL.

A segunda chave é usada para seleção em VARIABLE2 em uma entrada de roteamento da origem de dados. O local deve estar no formato InputBody ou InputRoot.
Gerenciador de filas padrão Não Sim

Este parâmetro define o gerenciador de filas para o qual as mensagens são enviadas quando os critérios para o valor da propriedade Consultar rotas não são atendidos.
Fila padrão Não Sim DEFAULT

Este parâmetro define a fila para a qual mensagens são enviadas quando os critérios para o valor da propriedade Consultar rotas não são atendidos.
Atualização de cache Não Não Limpo

Se este parâmetro for selecionado, quando uma mensagem de atualização for recebida o cache de dados de roteamento na memória será atualizado a partir da origem de dados.
Fila de atualização Não Sim REFRESH

Se Atualização de cache for selecionado, uma mensagem que é enviada para esta fila acionará uma atualização dos dados de roteamento na memória.
Criação de log
Parâmetro de padrão M C Default Descrição
Criação de log requerida Não Não Limpo

Este parâmetro padrão determina se a instância padrão inclui o código para criação de log.

Se Criação de log requerida estiver limpa, um subfluxo de Log não será incluído.

Se Criação de log requerida estiver selecionado, um subfluxo de Log será incluído no fluxo de mensagens e os valores deverão ser configurados para Gerenciador de filas de log e Fila de Logs.
Gerenciador de filas de log Não Sim

Este parâmetro padrão define o gerenciador de filas para criação de log. Ele poderá ser deixado em branco se o gerenciador de filas do broker for usado para criação de log.

Este parâmetro é necessário apenas se Criação de log requerida é selecionado.

Fila de Logs Não Sim LOG

Este parâmetro padrão define a fila para criação de log.

Este parâmetro é necessário apenas se Criação de log requerida é selecionado.
Tratamento de erro
Parâmetro de padrão M C Default Descrição
Mensagem de erro necessária Não Não Selecionado

Este parâmetro padrão define se o aplicativo padrão cria os elementos de fluxo de mensagens que criam as mensagens de erro.

Se Mensagem de erro requerida for selecionado, os valores deverão ser configurados para Gerenciador de filas de erros e Fila de erros.
Gerenciador de filas de erros Não Sim  

Este parâmetro padrão define o gerenciador de filas para mensagens de erro. Ele poderá ser deixado em branco se o gerenciador de filas do broker for usado para criação de log.
Fila de erros Não Sim ERROR

Este parâmetro padrão define a fila para mensagens de erro.

Este parâmetro é necessário apenas se Mensagem de erro requerida é selecionado.

Gerais
Parâmetro de padrão M C Default Descrição
Esquema do broker Não Não mqsi

Se um esquema do broker é especificado, todos os fluxos de mensagens e arquivos ESQL são criados nesse esquema. Se nenhum esquema for especificado, o esquema do broker padrão será usado.
Prefixo do fluxo Não Não

Se configurado, este prefixo é incluído no início de todos os nomes de fluxo de mensagens.
Sufixo do fluxo Não Não

Se configurado, este sufixo é incluído no final de todos os nomes de fluxo de mensagens.
Prefixo de Fila Não Não

Se configurado, este prefixo é incluído no início de todos os nomes da fila.

Se os nomes de fila padrão forem usados, o prefixo, o sufixo, ou ambos, deverão ser configurados para identificar nomes de fila para cada instância padrão.

Sufixo da Fila Não Não

Se configurado, este sufixo é incluído no final de todos os nomes de fila.

Se os nomes de fila padrão forem usados, o prefixo, o sufixo, ou ambos, deverão ser configurados para identificar nomes de fila para cada instância padrão.

Descrição curta Não Não

Esta descrição é incluída na propriedade Descrição Resumida de todos os fluxos de mensagens que são gerados.
Descrição detalhada Não Não

Esta descrição é incluída na propriedade Descrição Detalhada de todos os fluxos de mensagens que são gerados.

Todos os parâmetros padrão são listados com seus valores-padrão.

A coluna intitulada O indica se a propriedade é obrigatória. A coluna intitulada C indica se a propriedade é configurável, significando que é possível alterar o valor quando você inclui o fluxo de mensagens no arquivo broker archive (BAR).

Onde nenhum padrão é indicado, o valor-padrão é uma cadeia vazia.
