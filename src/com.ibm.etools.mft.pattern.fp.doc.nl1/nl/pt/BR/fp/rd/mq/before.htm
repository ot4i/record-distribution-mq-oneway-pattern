Tarefas para Concluir Antes de Aplicar o Padrão de Record Distribution para WebSphere MQ: Unidirecional

Antes de aplicar o padrão para produzir uma instância padrão, é necessário determinar quais das tarefas a seguir são necessárias. Se necessário, conclua as tarefas a seguir:

  • Identifique se este padrão atende aos requisitos de negócios e pode ser suportado pelos criadores de arquivos e pelos consumidores de mensagens.
  • Nomeie a instância padrão. Identifique o propósito do aplicativo padrão para que você possa decidir sobre um nome apropriado.
  • Decida se você requer a criação de log. Se a criação de log for necessária, é preciso decidir para qual gerenciador de filas e fila as mensagens de criação de log são enviadas. A criação de log pode ser para uma fila de logs gerais, que já existe para outros fluxos do broker ou para uma fila que seja criada para a instância padrão.
  • Decida se você requer mensagens de erro. Se mensagens de erro forem necessárias, você deverá decidir para qual gerenciador de filas e fila as mensagens de erro são enviadas.
  • Decida como os registros são detectados e, se necessário, prepare um conjunto de mensagens para definir como os registros são analisados.
  • Decida se a validação é necessária e, em caso afirmativo, prepare um conjunto de mensagens para definir como os registros são analisados.
  • Determine se os arquivos que devem ser processados são locais ou remotos. Se arquivos remotos precisarem ser processados, o nome do serviço configurável deverá ser identificado.
  • Decida se existem múltiplos destinos possíveis. Se múltiplos destinos possíveis existirem, selecione a abordagem de roteamento a usar.
    • Se você tiver um número limitado de rotas em que os destinos não são alterados frequentemente, configure o parâmetro Roteamento como Especificar Rotas.
    • Se o conjunto de rotas de destino não for determinado totalmente ou puder ser alterado frequentemente, configure o parâmetro Roteamento como Consultar Rotas. Se este valor for selecionado, o nome da origem de dados deverá ser conhecido.
