Restrições sobre o Uso do Padrão de Record Distribution para WebSphere MQ: Unidirecional

Várias restrições se aplicam quando este padrão é usado.

É possível usar o padrão de Record Distribution para WebSphere MQ: Unidirecional apenas se você atender a todas as restrições a seguir:

  • Deve ser possível ler o arquivo de entrada como registros. É possível processar arquivos que contêm registros de comprimento fixo, registros de comprimento variável separados por um delimitador bem conhecido ou registros de comprimento variável que podem ser extraídos por um analisador, uma técnica referida como Sequência do Registro Analisado. Esta técnica pode ser usada para ler um arquivo que contém múltiplos documentos XML, retornando cada documento como um registro. Não é possível usar este padrão para extrair registros integrados, por exemplo, registros em um único documento XML.
  • Se um arquivo remoto for processado pelo serviço FTP, o broker deverá ter permissão para excluir o arquivo de origem porque arquivos de origem são movidos para o broker e não são lidos no local.
  • A função de negócios para a qual a instância padrão é usada deve ser compatível com a redistribuição de registros após uma falha do sistema. Este padrão distribui os registros de maneira confiável, mas não protege contra duplicatas após uma falha do sistema.
  • A função de negócios, para a qual a instância padrão é usada, deve ser compatível com a redistribuição de registros após a falha para ler um registro do arquivo ou para gravar um registro que foi lido. Neste caso, este padrão move o arquivo inteiro para o diretório de restauração e um erro é gravado na fila de erros. É necessário organizar o arquivo para ser reenviado após o erro ser corrigido. Quando um arquivo é reenviado, todos os registros no arquivo são reprocessados, o que pode levar a mensagens duplicadas.
  • Se a criação de log for necessária, o formato que é usado por este padrão é assumido para atender aos requisitos de seu próprio ambiente.
