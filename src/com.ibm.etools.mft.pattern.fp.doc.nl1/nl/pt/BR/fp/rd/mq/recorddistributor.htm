Fluxo de Record Distributor para o Padrão de Record Distribution para WebSphere MQ: Unidirecional

Detalhes sobre o fluxo de Record distributor para o padrão de Record Distribution para WebSphere MQ: Unidirecional.

O fluxo executa as seguintes ações:

  1. Um arquivo é selecionado para processamento de acordo com os parâmetros de entrada de arquivo.
  2. Os registros são lidos a partir do arquivo de acordo com os parâmetros de detecção de registro. Cada registro é processado em uma transação. Se a validação for especificada, o registro será validado em um conjunto de mensagens.
  3. O subfluxo de Rota é chamado para determinar o destino correto do WebSphere MQ e o registro é gravado na fila de destino correta. Quando um registro é gravado em uma fila, por padrão ele está no mesmo formato em que foi lido a partir do arquivo. Se necessário, é possível transformar o registro em outro formato customizando o fluxo do Processador de Registro.
  4. Se Sem Roteamento for selecionado, o registro será gravado na fila de mensagens padrão.
  5. Se ocorrerem erros, por exemplo falha ao ler um registro ou em gravar um registro em uma fila, uma mensagem de erro será gravada e o fluxo será restaurado. O arquivo é gravado no diretório de restauração.
  6. Quando todos os dados forem processados, o subfluxo de Log será chamado para registrar a conclusão. Se quaisquer registros tiverem sido gravados na fila de mensagens padrão, esta ação será observada.
  7. O fluxo finaliza com êxito e o arquivo é movido para o subdiretório do archive.
