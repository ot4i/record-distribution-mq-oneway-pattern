Tarefas para Concluir Após Gerar o Padrão de Record Distribution para WebSphere MQ: Unidirecional

As tarefas que você deve concluir após ter gerado o padrão.

  1. Crie o arquivo broker archive (BAR). Inclua o fluxo de mensagens gerado em seu arquivo BAR. No editor do Broker Archive, você deve selecionar a opção Compilar e Sequenciar Recursos antes de construir o arquivo BAR. Se tiver usado um conjunto de mensagens para definir como os registros são analisados, você deverá incluir o conjunto de mensagens no arquivo BAR.
  2. Configure o arquivo broker archive. A configuração adicional do fluxo de mensagens no arquivo bar não é necessária antes de implementar os fluxos de mensagens para sua instância padrão de Record Distribution para WebSphere MQ: Unidirecional, mas talvez você deseje revisar algumas das opções padrão e configurar alterações para seu ambiente.

    Na página Gerenciar do editor de Broker Archive:

    • Opcional: Use a propriedade definida pelo usuário LoggingOn para selecionar se a criação de log está ativada ou desativada para o fluxo de mensagens.
    • Opcional: Use a propriedade definida pelo usuário ErrorLoggingOn para selecionar se as mensagens de erro são geradas.
    • Opcional: Também é possível alterar qualquer uma das propriedades configuráveis no broker archive, por exemplo, propriedades para segurança ou criar instâncias adicionais. É necessário assegurar que quaisquer valores que você alterar não causem impacto no comportamento da instância padrão.
  3. No arquivo de resumo, visualize a lista de filas que você deve criar. O arquivo de resumo está localizado na pasta Configuração de Padrão do projeto de instância padrão InstanceName.
  4. Crie as filas. Quando aplicar o padrão, uma lista de filas do WebSphere MQ será gerada com nomes de filas e nomes de gerenciador de filas, estas filas do WebSphere MQ deverão ser criadas se ainda não existirem. Todas as configurações de mensagem que são necessárias por padrão são configuradas explicitamente nas mensagens e não são dependentes das configurações de filas.
  5. Crie os recursos do WebSphere MQ. É possível criar recursos do WebSphere MQ na linha de comandos, usando programas e utilitários como runmqsc, ou usando a interface gráfica do WebSphere MQ Explorer. Para obter informações adicionais sobre a criação de recursos do WebSphere MQ, consulte WebSphere MQ Intercommunication, disponível na página da Web da biblioteca do WebSphere MQ.
  6. Se usar o valor de Rotas de Consulta no parâmetro Roteamento, você poderá definir rotas dinamicamente no banco de dados mas as rotas não serão listadas no arquivo de resumo. Se você tiver incluído rotas no banco de dados, deverá criar as filas e gerenciadores de filas correspondentes.
  7. Configure um banco de dados de roteamento se necessário. Se o valor Rotas de Consulta for selecionado no parâmetro Roteamento, a origem de dados deverá ser configurada com uma tabela ROUTING_TABLE. Esta tabela deve incluir as seguintes colunas:
    • ROUTING_CONTEXT

      Use um valor que identifica as entradas de roteamento para este fluxo.

    • VARIABLE1

      O valor ao qual a propriedade KEY1 do elemento de mensagem deve corresponder.

    • VARIABLE2

      O valor ao qual a propriedade KEY2 do elemento de mensagem deve corresponder.

    • QUEUE_MANAGER

      O gerenciador de filas do destino.

    • QUEUE_NAME

      A fila de destino.

  8. Se tiver escolhido usar FTP para obter arquivos de entrada, você deverá criar um serviço configurável FtpServer no broker de destino correspondente ao nome especificado em seu padrão. Esta ação é requerida pelo nó FileInput para conectar-se ao servidor FTP remoto.
  9. Implemente o arquivo BAR. Se um conjunto de mensagens for usado para definir registros, este conjunto de mensagens deverá ser implementado usando o mesmo ou outro arquivo BAR.
