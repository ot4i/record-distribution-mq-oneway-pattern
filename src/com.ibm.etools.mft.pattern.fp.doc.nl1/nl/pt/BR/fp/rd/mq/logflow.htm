Subfluxo de Log para o Padrão de Record Distribution para WebSphere MQ: Unidirecional

A criação de log é controlada pela propriedade definida pelo usuário: LoggingOn.

As mensagens de log são gravadas como mensagens do WebSphere MQ persistentes em uma fila do WebSphere MQ. Estas mensagens de log podem ser processadas separadamente de acordo com as práticas padrão em sua organização.

Os detalhes a seguir sobre o contexto estão contidos no cabeçalho MQRFH2 da mensagem que é gravada na fila de logs:

  • Nome do broker
  • Nome do fluxo de mensagens
  • Registro de data e hora

Os detalhes a seguir sobre o arquivo e registros estão contidos no corpo XML da mensagem que é gravada na fila de log:

  • Diretório
  • Nome do arquivo
  • Total de registros
  • Registros no padrão

Uma mensagem de log é enviada à fila de logs apenas quando um arquivo inteiro é concluído com êxito. Qualquer processamento de arquivo, que causa um erro e retira o arquivo, gera uma mensagem de erro contendo detalhes de onde o erro ocorreu.
