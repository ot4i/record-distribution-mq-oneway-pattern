Tasks, die nach der Generierung des Musters Record Distribution to WebSphere MQ: one-way (Verteilung von Datensätzen an WebSphere MQ: unidirektional) ausgeführt werden müssen

Tasks, die Sie nach der Generierung des Musters ausführen müssen.

  1. Erstellen Sie die Brokerarchivdatei (BAR-Datei). Fügen Sie den generierten Nachrichtenfluss Ihrer BAR-Datei hinzu. Im Brokerarchiveditor müssen Sie die Option Ressourcen kompilieren und integrieren auswählen, um die BAR-Datei zu erstellen. Wenn Sie einen Nachrichtensatz zur Definition der Parsing-Weise für Datensätze verwendet haben, müssen Sie der Brokerarchivdatei den Nachrichtensatz hinzufügen.
  2. Konfigurieren Sie die Brokerarchivdatei. Eine weitere Konfiguration des Nachrichtenflusses in der Brokerarchivdatei ist nicht erforderlich, bevor Sie die Nachrichtenflüsse für Ihre Instanz des Musters Record Distribution to WebSphere MQ: one-way (Verteilung von Datensätzen an WebSphere MQ: unidirektional) implementieren, Sie können jedoch einige Standardoptionen überprüfen und Änderungen für Ihre Umgebung konfigurieren.

    Auf der Seite Verwalten des Editors Brokerarchiv können Sie folgende Vorgänge ausführen:

    • Optional: Wählen Sie mithilfe der benutzerdefinierten Eigenschaft LoggingOn aus, ob die Protokollierung für den Nachrichtenfluss aktiviert oder inaktiviert ist.
    • Optional: Wählen Sie mithilfe der benutzerdefinierten Eigenschaft ErrorLoggingOn aus, ob Fehlernachrichten generiert werden.
    • Optional: Sie können auch die konfigurierbaren Eigenschaften im Brokerarchiv ändern, zum Beispiel Sicherheitseigenschaften oder Eigenschaften zur Erstellung weiterer Instanzen. Allerdings müssen Sie sicherstellen, dass sich die von Ihnen geänderten Werte nicht auf das Verhalten der Musterinstanz auswirken.
  3. Die zu erstellenden Warteschlangen entnehmen Sie bitte der Zusammenfassungsdatei. Die Summendatei befindet sich im Projektverweisordner des Musterinstanzprojektes Instanzname.
  4. Erstellen Sie die Warteschlangen. Bei der Anwendung des Musters wird eine Liste mit WebSphere MQ-Warteschlangen generiert. Diese Liste enthält die Namen von Warteschlangen und Warteschlangenmanagern. Falls diese WebSphere MQ-Warteschlangen noch nicht vorhanden sind, müssen sie erstellt werden. Alle Nachrichteneinstellungen, die für das Muster erforderlich sind, werden explizit in den Nachrichten festgelegt und hängen nicht von Warteschlangeneinstellungen ab.
  5. Erstellen Sie die WebSphere MQ-Ressourcen. Sie können WebSphere MQ-Ressourcen entweder über die Befehlszeile mithilfe von Programmen und Dienstprogrammen wie runmqsc oder mithilfe der grafischen Oberfläche von WebSphere MQ Explorer erstellen. Weitere Informationen zur Erstellung von WebSphere MQ-Ressourcen finden Sie unter WebSphere MQ Intercommunication, verfügbar auf der WebSphere MQ-Bibliothekswebseite.
  6. Wenn Sie den Wert Routen suchen im Parameter Routing verwenden, können Sie Routen dynamisch in der Datenbank definieren, die Routen werden jedoch nicht in der Summendatei aufgelistet. Wenn Sie Routen zur Datenbank hinzugefügt haben, müssen Sie die entsprechenden Warteschlangen und WS-Manager erstellen.
  7. Richten Sie bei Bedarf eine Routing-Datenbank ein. Wenn der Wert Routen suchen im Parameter Routing ausgewählt ist, muss die Datenquelle mit der Tabelle 'ROUTING_TABLE' konfiguriert werden. Diese Tabelle muss die folgenden Spalten enthalten:
    • ROUTING_CONTEXT

      Geben Sie einen Wert ein, der die Routing-Einträge für diesen Nachrichtenfluss identifiziert.

    • VARIABLE1

      Der Wert, mit dem die Eigenschaft KEY1 des Nachrichtenelements übereinstimmen muss.

    • VARIABLE2

      Der Wert, mit dem die Eigenschaft KEY2 des Nachrichtenelements übereinstimmen muss.

    • QUEUE_MANAGER

      Der Warteschlangenmanager des Ziels.

    • QUEUE_NAME

      Die Zielwarteschlange.

  8. Wenn Sie FTP zum Abruf von Eingabedateien nutzen möchten, müssen einen konfigurierbaren FtpServer-Service auf dem Zielbroker erstellen, der dem Namen entspricht, der in Ihrem Muster angegeben ist. Diese Aktion ist erforderlich, damit sich der FileInput-Knoten mit dem fernen FTP-Server verbinden kann.
  9. Implementieren Sie die BAR-Datei. Wenn ein Nachrichtensatz für die Definition von Datensätzen verwendet wird, muss dieser Nachrichtensatz mithilfe derselben oder einer anderen BAR-Datei implementiert werden.
