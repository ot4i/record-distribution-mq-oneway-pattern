Tasks, die vor der Anwendung des Musters Record Distribution to WebSphere MQ: one-way (Verteilung von Datensätzen an WebSphere MQ: unidirektional) ausgeführt werden müssen.

Bevor Sie das Muster für die Erstellung einer Musterinstanz anwenden, müssen Sie bestimmen, welche der folgenden Tasks erforderlich sind. Führen Sie folgende Tasks aus, falls erforderlich:

  • Stellen Sie fest, ob dieses Muster den Geschäftsanforderungen entspricht und von Dateierstellern und Nachrichtenkonsumenten unterstützt werden kann.
  • Benennen Sie die Musterinstanz. Bestimmen Sie den Zweck der Musteranwendung, sodass Sie einen geeigneten Namen festlegen können.
  • Entscheiden Sie, ob eine Protokollierung erforderlich ist. Falls ja, bestimmen Sie den Warteschlangenmanager und die Warteschlange für die Protokollnachrichten. Die Protokollierung kann aber auch in eine allgemeine Warteschlange, die bereits für andere Brokernachrichtenflüsse vorhanden ist, oder in eine Warteschlange, die für die Musterinstanz erstellt wird, erfolgen.
  • Entscheiden Sie, ob Fehlernachrichten erforderlich sind. Ist dies der Fall, müssen Sie festlegen, an welchen WS-Manager und an welche Warteschlange die Fehlernachrichten gesendet werden.
  • Entscheiden Sie, wie Datensätze erkannt werden sollen, und bereiten Sie gegebenenfalls einen Nachrichtensatz vor, um zu definieren, wie Datensätze syntaktisch analysiert werden.
  • Entscheiden Sie, ob eine Auswertung erforderlich ist, und bereiten Sie gegebenenfalls einen Nachrichtensatz vor, um zu definieren, wie Datensätze syntaktisch analysiert werden.
  • Bestimmen Sie, ob die Dateien, die verarbeitet werden sollen, lokal oder fern sind. Wenn ferne Dateien verarbeitet werden sollen, muss der Name des konfigurierbaren Services identifiziert werden.
  • Bestimmen Sie, ob mehrere mögliche Ziele vorhanden sind. Ist dies der Fall, wählen Sie die Routing-Methode aus, die verwendet werden soll.
    • Wenn eine begrenzte Anzahl von Routen vorhanden ist und sich die Ziele nicht häufig ändern, setzen Sie den Parameter Routing auf Routen angeben.
    • Wenn die Gruppe der Zielrouten nicht vollständig bestimmt ist oder sich häufig ändern kann, setzen Sie den Parameter Routing auf Routen suchen. Bei dieser Einstellung muss der Name der Datenquelle bekannt sein.
