Datensatzverteiler-Nachrichtenfluss für das Muster Record Distribution to WebSphere MQ: one-way (Verteilung von Datensätzen an WebSphere MQ: unidirektional)

Informationen zum Datensatzverteiler-Nachrichtenfluss für das Muster Record Distribution to WebSphere MQ: one-way (Verteilung von Datensätzen an WebSphere MQ: unidirektional).

Der Nachrichtenfluss führt folgende Aktionen aus:

  1. Eine Datei wird entsprechend den Dateieingabeparametern zur Verarbeitung ausgewählt.
  2. Datensätze werden entsprechend der Datensatzerkennungsparameter aus der Datei gelesen. Jeder Datensatz wird in einer Transaktion verarbeitet. Wenn eine Auswertung angegeben ist, wird der Datensatz auf Basis des Nachrichtensatzes ausgewertet.
  3. Der untergeordnete Routing-Nachrichtenfluss wird aufgerufen, um das korrekte WebSphere MQ-Ziel zu bestimmen, und der Datensatz wird in die korrekte Zielwarteschlange geschrieben. Wenn ein Datensatz in eine Warteschlange geschrieben wird, hat er standardmäßig dasselbe Format wie beim Lesen aus der Datei. Sie können den Datensatz gegebenenfalls in ein anderes Format umwandeln, indem Sie den Datensatzprozessor-Nachrichtenfluss anpassen.
  4. Wenn Kein Routing ausgewählt ist, wird der Datensatz in die Standardnachrichtenwarteschlange geschrieben.
  5. Wenn Fehler auftreten, beispielsweise weil ein Datensatz nicht gelesen oder in eine Warteschlange geschrieben werden kann, wird eine Fehlernachricht geschrieben und der Nachrichtenfluss wird zurückgesetzt. Die Datei wird in das Verzeichnis für die Zurücksetzung geschrieben.
  6. Wenn alle Daten verarbeitet sind, wird der untergeordnete Protokollnachrichtenfluss aufgerufen, um den Abschluss aufzuzeichnen. Wenn Datensätze in die Standardnachrichtenwarteschlange geschrieben wurden, wird diese Aktion vermerkt.
  7. Der Nachrichtenfluss wird erfolgreich beendet und die Datei wird in das Archivunterverzeichnis verschoben.
