Tareas que deben completarse antes de aplicar el patrón Distribución de registros en WebSphere MQ: unidireccional

Antes de aplicar el patrón para producir una instancia de patrón, debe determinar cuáles de las tareas siguientes son necesarias. Si es preciso, realice las tareas siguientes:

  • Identifique que este patrón cumple los requisitos empresariales y puede estar soportado por creadores de archivos y consumidores de mensajes.
  • Asigne un nombre a la instancia de patrón. Identifique el fin de la aplicación de patrón para que pueda decidir cuál es el nombre adecuado.
  • Decida si necesita un registro. Si el registro es necesario, debe decidir a qué gestor de colas y cola se enviarán los mensajes de registro. El registro puede efectuarse en una cola de registro general, que ya exista para otros flujos de intermediario o en una cola que se haya creado para la instancia del patrón.
  • Decida si necesita mensajes de error. Si los mensajes de error son necesarios, debe decidir a qué gestor de colas y cola se enviarán los mensajes de error.
  • Decida cómo se detectarán los registros y, si es necesario, prepare un conjunto de mensajes para definir cómo se analizarán los registros.
  • Decidir si la validación es necesaria y, si lo es, prepare un conjunto de mensajes para definir cómo se analizarán los registros.
  • Determine si los archivos que se van a procesar son locales o remotos. Si se van a procesar archivos remotos, es preciso identificar el nombre del servicio configurable.
  • Decida si hay varios destinos posibles. Si hay varios destinos posibles, seleccione el enfoque de direccionamiento que va a utilizar.
    • Si tiene un número limitado de rutas en que los destinos no cambian con frecuencia, establezca el parámetro Direccionamiento en Especificar rutas.
    • Si el conjunto de rutas de destino no está totalmente determinado o puede cambiar con frecuencia, establezca el parámetro Direccionamiento en Buscar rutas. Si este valor es seleccionado, el nombre del origen de datos debe ser conocido.
