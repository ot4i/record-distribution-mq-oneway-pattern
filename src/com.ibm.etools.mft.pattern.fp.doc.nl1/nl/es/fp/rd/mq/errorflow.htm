Subflujo Error para el patrón Distribución de registros en WebSphere MQ: unidireccional

Este subflujo prueba si debe escribirse un mensaje de error y lo determina la propiedad definida por el usuario: ErrorLoggingOn.

Si se detecta un error durante el proceso de flujos de mensajes, se llamará al subflujo Error. El subflujo Error realiza las acciones siguientes:

  1. Si los mensajes de error son necesarios, se preparará un mensaje de error y se grabará en la cola de errores. Los detalles de la excepción se graban como un mensaje XML. Estos detalles incluyen:
    • Nombre del intermediario
    • Nombre de flujo
    • Indicación horaria
    • Directorio de archivos
    • Nombre de archivo
    • Número de registro, cuando se ha producido un error
    • Número de registros grabados en la cola predeterminada
    • Resumen de los datos de excepción

    El resumen de los datos de excepción contienen la etiqueta del flujo de mensajes, el número del error, la descripción del texto y una lista de inserciones que ofrecen más detalles.

  2. Se produce una excepción que hace que el flujo finalice de manera incorrecta. Esta acción mueve el archivo al subdirectorio de retrotracción.
Nota: Los mensajes que no se pueden direccionar se escriben en un destino predeterminado. Esta acción no se trata como un error.

Una anomalía que detiene un registro que se está leyendo desde el archivo y se está grabando en el disco se trata como si fuera un error.
