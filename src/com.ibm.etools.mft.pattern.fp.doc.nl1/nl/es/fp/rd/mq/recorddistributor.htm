Flujo Distribución de registros para el patrón Distribución de registros en WebSphere MQ: unidireccional

Detalles sobre el flujo Distribución de registros para el patrón Distribución de registros en WebSphere MQ: unidireccional.

El flujo realiza las acciones siguientes:

  1. Se selecciona un archivo para un proceso de acuerdo con los parámetros de entrada de archivos.
  2. Los registros se leen del archivo de acuerdo con los parámetros de detección de registros. Cada registro se procesa dentro de una transacción. Si se especifica la validación, el registro se valida con un conjunto de mensajes.
  3. Se llama al subflujo Route para determinar el destino correcto de WebSphere MQ y el registro se escribe en la cola de destino correcta. Cuando un registro se escribe en una cola, está de forma predeterminada el mismo formato como se ha leído del archivo. Si es necesario, puede transformar el registro a otro formato personalizando el flujo Record Processor.
  4. Si se selecciona Sin direccionamiento, el registro se escribe en la cola de mensajes predeterminada.
  5. Si se producen errores, por ejemplo, que no se pueda leer un registro o grabar un registro en una cola, se graba un mensaje de error y se retrotrae el flujo. El archivo se graba en el directorio de retrotracción.
  6. Cuando se procesan todos los datos, se llama al subflujo Log para completar el registro. Si se han escrito registros en la cola de mensajes predeterminada, se indica esta acción.
  7. El flujo finaliza correctamente y el archivo se mueve al subdirectorio de archivado.
