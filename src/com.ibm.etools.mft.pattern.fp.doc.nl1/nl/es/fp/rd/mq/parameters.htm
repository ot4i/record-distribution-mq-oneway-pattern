Parámetros para el patrón Distribución de registros en WebSphere MQ: unidireccional

Archivo de entrada
Parámetro de patrón O C Predeterminado Descripción
Directorio de archivo de entrada Sí Sí  

La ubicación del archivo de entrada.
Patrón de archivo Sí Sí *

Nombre de archivo o serie que contiene caracteres comodín opcionales (asterisco (*) o interrogante (?)) que identifican el archivo o archivos que se van a procesar del directorio de entrada.
Usar FTP Sí Sí Borrado

Este parámetro determina si se utiliza FTP para recuperar archivos remotos.
Servicio configurable FTP No Sí ftpservice

Este servicio configurable define cómo se utiliza FTP para recuperar un archivo remoto.
Proceso de registros de entrada
Parámetro de patrón O C Predeterminado Descripción
Detección de registros No No Longitud fija (binario o texto)

Los registros se leen del archivo de acuerdo con el método de detección de registros que se selecciona.

Los valores válidos son:

  • Longitud fija (binario o texto)
  • Delimitados (binario o texto)
  • Delimitados (XML)
  • Analizados (binario o texto)
  • Analizados (XML)

Si se selecciona Analizados (binario o texto), también deben configurarse los parámetros Conjunto de mensajes de entrada, Tipo de mensaje de entrada y Formato de mensaje de entrada.
Longitud de registro No No 80

Los registros de la longitud necesaria se leen del archivo de origen.
Tipo de delimitador No No Fin de línea de DOS o UNIX

Si los datos no utilizan el valor predeterminado, seleccione Delimitador personalizado (hexadecimal).
Delimitador de registro No No  

Los registros que se leen desde el archivo están separados por el delimitador especificado, que se proporciona en formato hexadecimal.

Este delimitador debe especificarse como un número par de hasta 32 caracteres, cada uno de ellos entre 0-9 o A-F (en mayúsculas o minúsculas).
Conjunto de mensajes de entrada No No  

Los registros se leen del archivo según una definición de este conjunto de mensajes.
Tipo de mensaje de entrada No No  

Los registros se leen y se analizan según este tipo de mensajes.
Formato de mensaje de entrada No No  

Los registros se leen y se analizan según este formato de mensajes.
CCSID de archivo Sí Sí Valor predeterminado del sistema de intermediario

Este parámetro define el conjunto de caracteres codificados que se utiliza para interpretar bytes del archivo que se lee.
Codificación de archivo No Sí Determinado por el sistema de intermediario

Este parámetro define el esquema de codificación para los números y caracteres grandes que se utilizan para interpretar bytes del archivo que se lee. Los valores válidos son:

  • Determinado por el sistema de intermediario
  • Little endian, con coma flotante IEEE (546)
  • Big endian, con coma flotante IEEE (273)
  • Big endian, con coma flotante S390 (785)

Para obtener información sobre la codificación, consulte "Conversión de datos con flujos de mensajes" en la documentación de IBM Integration.
Direccionamiento
Parámetro de patrón O C Predeterminado Descripción
Direccionamiento No No Sin direccionamiento

Este parámetro define el método de direccionamiento que se va a utilizar. Los valores válidos son:

  • Sin direccionamiento
  • Especificar rutas
  • Buscar rutas
Sin direccionamiento
Parámetro de patrón O C Predeterminado Descripción
Gestor de colas de salida No Sí

Este parámetro define el gestor de colas al que se envían todos los mensajes de salida.

Cola de salida No Sí OUTPUT

Este parámetro define la cola a la que se envían todos los mensajes de salida.

Especificar rutas
Parámetro de patrón O C Predeterminado Descripción
Gestor de colas predeterminado No Sí

Este parámetro define el gestor de colas al que se envían los mensajes cuando no se cumplen los criterios para el valor de propiedad Especificar rutas.

Cola predeterminada No Sí DEFAULT

Este parámetro define la cola a la que se envían los mensajes cuando no se cumplen los criterios para el valor de propiedad Especificar rutas.

Los parámetros de la Tabla de espacio de nombres siguiente definen los prefijos de espacio de nombres y los URI de espacio de nombres que puede utilizar en las expresiones XPath que definen las ubicaciones de clave para rutas. Si se incluye una fila, deben incluirse ambos parámetros.

Puede añadir, editar y suprimir valores y cambiar el orden de los valores en la tabla.

Parámetro Tabla de espacio de nombres O C Predeterminado Descripción
Prefijo de espacio de nombres Sí No

Este parámetro define el código para un espacio de nombres.
URI de espacio de nombres Sí No

Este parámetro define el espacio de nombres para el código actual.

La Tabla de espacio de nombres define los espacios de nombres que están disponibles para utilizar en expresiones XPath en la Tabla de direccionamiento.

Parámetro Tabla de direccionamiento O C Predeterminado Descripción
Ubicación de claves Sí No

Una ubicación de clave es una expresión XPath que define una ubicación en el registro que se está propagando. Por ejemplo, $Root/XMLNSC/submitPORequest/partNo.

Para obtener más información sobre el tipo de expresión que se debe utilizar, consulte "Nodo Route" y "Utilización de XPath" en la documentación de IBM Integration.

Los espacios de nombres pueden incluirse si se especifican en la tabla de espacio de nombres opcional.

Valor de clave Sí No

Este parámetro contiene el valor con el que el campo Ubicación de claves del mensaje entrante debe coincidir para seleccionar esta ruta.

Gestor de colas No Sí

Este parámetro define el gestor de colas para un destino de direccionamiento.

Cola Sí Sí

Este parámetro define la cola para un destino de direccionamiento.

Buscar rutas
Parámetro de patrón O C Predeterminado Descripción
Origen de datos de direccionamiento No Sí

Este parámetro define el origen de datos desde el que se puede seleccionar una ruta. La información de direccionamiento se busca en la tabla ROUTING_TABLE.

Contexto de datos de direccionamiento No No Este parámetro define un contexto de direccionamiento de manera que puede utilizar una ROUTING_TABLE para más de un flujo de mediación.

Sólo se seleccionan las filas en las que el campo ROUTING_CONTEXT coincide con el valor de Contexto de datos de direccionamiento para la búsqueda de direccionamiento y se añaden a la memoria caché en memoria.

Ubicación de claves 1 No No

Este parámetro define cómo se extrae la primera clave del elemento de mensajes propagado. Se define como una expresión ESQL.

La primera clave se utiliza para seleccionar en VARIABLE1 en una entrada de direccionamiento de origen de datos. La ubicación debe estar en el formato InputBody o InputRoot.
Ubicación de claves 2 No No

Este parámetro define cómo una segunda clave opcional se extrae del elemento de mensaje propagado. Se define como una expresión ESQL.

La segunda clave se utiliza para seleccionar en VARIABLE2 en una entrada de direccionamiento de origen de datos. La ubicación debe estar en el formato InputBody o InputRoot.
Gestor de colas predeterminado No Sí

Este parámetro define el gestor de colas al que se envían los mensajes cuando no se cumplen los criterios para el valor de propiedad Buscar rutas.

Cola predeterminada No Sí DEFAULT

Este parámetro define la cola a la que se envían los mensajes cuando no se cumplen los criterios para el valor de propiedad Buscar rutas.

Renovación de memoria caché No No Borrado

Si este parámetro está seleccionado, cuando se recibe un mensaje de renovación, la memoria caché en memoria de datos de direccionamiento se renueva desde el origen de datos.

Renovar cola No Sí REFRESH

Si Renovación de memoria caché está seleccionado, un mensaje que se envíe a esta cola desencadena la renovación de los datos de direccionamiento en memoria.

Registro
Parámetro de patrón O C Predeterminado Descripción
Registro obligatorio No No Borrado

Este parámetro de patrón determina si la instancia de patrón incluye el código de registro.

Si se borra Registro necesario, no se incluye un subflujo Log.

Si se selecciona Registro obligatorio, se incluye un subflujo Log en el flujo de mensajes, y se deben establecer los valores para Gestor de colas de registro y Cola de registro.

Gestor de colas de registro No Sí

Este parámetro de patrón define el gestor de colas para el registro. Se puede dejar en blanco si se utiliza el gestor de colas de intermediario para el registro.

Este parámetro es necesario sólo si se ha seleccionado Registro necesario.

Cola de registro No Sí LOG

Este parámetro de patrón define la cola de registro.

Este parámetro es necesario sólo si se ha seleccionado Registro necesario.

Manejo de errores
Parámetro de patrón O C Predeterminado Descripción
Mensaje de error obligatorio No No Seleccionado

Este parámetro de patrón define si la aplicación de patrón crea los elementos de flujo de mensajes que crean los mensajes de error.

Si Mensaje de error necesario está seleccionado, deben establecerse valores para Gestor de colas de errores y Cola de errores.
Gestor de colas de errores No Sí  

Este parámetro de patrón define el gestor de colas para los mensajes de error. Se puede dejar en blanco si se utiliza el gestor de colas de intermediario para el registro.

Cola de errores No Sí ERROR

Este parámetro de patrón define la cola para mensajes de error.

Este parámetro es necesario sólo si se ha seleccionado Mensaje de error necesario.

General
Parámetro de patrón O C Predeterminado Descripción
Esquema de intermediario No No mqsi

Si se especifica un esquema de intermediario, todos los flujos de mensajes y archivos ESQL se crean en ese esquema. Si no se especifica ningún esquema, se utiliza el esquema de intermediario predeterminado.

Prefijo de flujo No No

Si se establece, este prefijo se añade al inicio de todos los nombres de flujo de mensajes.

Sufijo de flujo No No

Si se establece, este sufijo se añade al final de todos los nombres de flujo de mensajes.

Prefijo de cola No No

Si se establece, este prefijo se añade al inicio de todos los nombres de cola.

Si se utilizan nombres de cola predeterminados, debe establecerse el prefijo, el sufijo, o ambos para identificar los nombres de cola para cada instancia de patrón.

Sufijo de cola No No

Si se establece, este sufijo se añade al final de todos los nombres de cola.

Si se utilizan nombres de cola predeterminados, debe establecerse el prefijo, el sufijo, o ambos para identificar los nombres de cola para cada instancia de patrón.

Descripción corta No No

Esta descripción se añade a la propiedad Descripción corta de todos los flujos de mensajes que se generan.

Descripción larga No No

Esta descripción se añade a la propiedad Descripción larga de todos los flujos de mensajes que se generan.

Todos los parámetros de patrón aparecen en la lista con sus valores predeterminados.

La columna con la cabecera O indica si la propiedad es obligatoria. La columna con la cabecera C indica si la propiedad es configurable, lo que quiere decir que puede cambiar el valor al añadir el flujo de mensajes al archivo archivador de intermediario (BAR).

Si no se indica el valor predeterminado, éste es una serie vacía.
